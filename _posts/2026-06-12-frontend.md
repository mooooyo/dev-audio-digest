# 2026-06-12 frontend 뉴스 정리

## Next.js

Next.js v16.3.0-canary.49 프리리지를 출시했습니다. 이 버전은 [turbopack](https://github.com/vercel/next.js/pull/94625)에서 변수 이름을 변경하고 테스트를 추가하는 등 여러 변경 사항을 포함합니다. 또한 개발 중에 부분적 미리 가져오기(partial prefetching) 없이 `prefetch={true}` 네비게이션에 경고 메시지를 표시하는 기능이 추가되었습니다. ([#94672](https://github.com/vercel/next.js/pull/94672))

다음은 전체 변경 사항 목록입니다:

* [turbopack] `path_join`에서 변수 이름 변경 및 테스트 추가: [#94625](https://github.com/vercel/next.js/pull/94625)
* 개발 중 부분적 미리 가져오기 없이 `prefetch={true}` 네비게이션에 경고 표시: [#94672](https://github.com/vercel/next.js/pull/94672)
* 개발 서버에서 만료될 때까지 캐시된 항목을 제공함: [#94662](https://github.com/vercel/next.js/pull/94662)
* 부분적 미리 가져오기가 활성화된 경우 동적 콘텐츠를 네비게이션 시 다시 가져옴: [#94655](https://github.com/vercel/next.js/pull/94655)
* 캐시 컴포넌트 마이그레이션 가이드 문서 확장: [#94649](https://github.com/vercel/next.js/pull/94649)
* `prefetch={true}` 네비게이션에 부분적 미리 가져오기가 없는 경우 소유자 스택 추가: [#94683](https://github.com/vercel/next.js/pull/94683)
* [turbopack] `rcstr!`를 사용하여 산재 수집 마이그레이션: [#94498](https://github.com/vercel/next.js/pull/94498)
* 사용자 영역 헤더에서 내부 개발 요청 ID 헤더 제거: [#94703](https://github.com/vercel/next.js/pull/94703)
* 개발 중 `'use cache: private'` 항목 지속: [#94694](https://github.com/vercel/next.js/pull/94694)
* [앱 셸] 즉각적인 유효성 검사를 더容易하게 membuat 새로운 단계 추가: [#94711](https://github.com/vercel/next.js/pull/94711)
* 터보 태스크를 산재 수집으로 마이그레이션: [#94503](https://github.com/vercel/next.js/pull/94503)
* [CC] 매개변수/search 매개변수에서 단계화된 렌더링 코드 경로 재구성: [#94718](https://github.com/vercel/next.js/pull/94718)
* [앱 셸] 셸 prefetch가 세션 데이터를 사용하는지 추적: [#94484](https://github.com/vercel/next.js/pull/94484)
* 빈 재개 본문을 동적 렌더링 요청으로 처리: [#94729](https://github.com/vercel/next.js/pull/94729)
